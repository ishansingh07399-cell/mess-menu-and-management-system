# 🍲 Smart AI Mess | Premium Dining Ecosystem

A high-performance, aesthetically driven web application designed to modernize the campus dining experience. **Smart AI Mess** combines real-time data visualization, AI-driven feedback loops, and a sleek glassmorphic UI to bridge the gap between students and mess management.

## 🚀 Key Features

### 1. Intelligent Dashboard
* **Live Attendance Tracking:** Reduces food waste by allowing students to mark "Eat" or "Skip" in real-time.
* **Dynamic Dietary Polls:** Visualizes the student body's preference (Veg vs. Non-Veg) using interactive progress bars.
* **Financial Insights:** Tracks current monthly billing with percentage shifts.

### 2. Premium Experience
* **3D Atomic Background:** A custom Three.js rendering engine providing a parallax-responsive particle field and atomic core structures.
* **Zomato-Style Menu:** A visually rich weekly menu curation with calorie counts and AI-generated star ratings.
* **Glassmorphic Design:** A modern dark-mode interface utilizing high-blur backdrops and vibrant brand gradients.

### 3. Management & Governance
* **AI Feedback Engine:** Sentiment-based feedback submission for meal quality.
* **Issue Reporting System:** A dedicated portal for reporting hygiene or quality issues with evidence attachment capabilities.
* **Mess Change Portal:** A structured window for monthly mess transfers.

## 🛠️ Tech Stack

* **Frontend:** HTML5, CSS3 (Custom Variables & Grid)
* **Animations:** [GSAP](https://greensock.com/gsap/) (ScrollTrigger & Timelines)
* **3D Engine:** [Three.js](https://threejs.org/) (WebGL)
* **Data Viz:** [Chart.js](https://www.chartjs.org/)
* **Icons:** FontAwesome 6

## 📦 Installation & Setup

1.  **Clone the repository:**
    ```bash
    git clone [https://github.com/yourusername/smart-ai-mess.git](https://github.com/yourusername/smart-ai-mess.git)
    ```
2.  **Open the project:**
    Simply open `index.html` in any modern web browser.
3.  **Local Storage:**
    The application uses `localStorage` to persist user profiles, feedback, and preferences without the need for a database during the frontend demo phase.
